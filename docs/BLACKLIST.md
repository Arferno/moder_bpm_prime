# Как работает чёрный список

## 🎯 Принцип работы

Бот проверяет **каждое текстовое сообщение** в группах/супергруппах на наличие запрещённых слов.

### Нормализация текста
Перед проверкой текст проходит через `normalize_for_blacklist()`:

```python
def normalize_for_blacklist(text: str) -> str:
    # 1. Удаляет все разделители между символами
    # "п р и в е т" → "привет"
    # "п.р.и.в.е.т" → "привет"
    # "п-р-и-в-е-т" → "привет"
    
    # 2. Приводит к нижнему регистру
    # "ПРИВЕТ" → "привет"
    # "ПрИвЕт" → "привет"
    
    # 3. Оставляет только буквы и цифры (Unicode)
    # "п@р#и$в%е^т" → "привет"
```

### Что ловится
| Исходное сообщение | Нормализованное | Сработает на "мат" |
|-------------------|-----------------|-------------------|
| `мат` | `мат` | ✅ |
| `МАТ` | `мат` | ✅ |
| `Мат` | `мат` | ✅ |
| `м а т` | `мат` | ✅ |
| `м.а.т` | `мат` | ✅ |
| `м-а-т` | `мат` | ✅ |
| `м_а_т` | `мат` | ✅ |
| `м*а*т` | `мат` | ✅ |

### Что НЕ ловится (частичные совпадения)
Слово должно встречаться как **подстрока** в нормализованном тексте, но алгоритм не делает морфологический разбор:

| Сообщение | Нормализованное | Сработает на "мат" |
|-----------|-----------------|-------------------|
| `математика` | `математика` | ❌ (это другое слово) |
| `материал` | `материал` | ❌ |
| `заматюк` | `заматюк` | ❌ |

> **Важно**: Если нужно ловить части слова — используй regex pattern при добавлении.

## ⚙️ Настройка действий

При добавлении слова указывается действие:

| Действие | Что делает | Длительность |
|----------|------------|--------------|
| `delete` | Только удаляет сообщение | — |
| `warn` | Удаляет + выдает варн (3 = бан) | — |
| `mute` | Удаляет + мут | Указана в секундах |
| `ban` | Удаляет + бан | Указана в секундах (0 = навсегда) |

### Примеры добавления
```
/blacklist add реклама mute 1h       # Мут 1 час за рекламу
/blacklist add спам warn             # Варн за спам
/blacklist add мат delete            # Только удалить мат
/blacklist add ссылка delete         # Удалять сообщения со ссылками
/blacklist add казино ban 7d         # Бан на неделю за казино
```

## 🔧 Regex-паттерны (продвинуто)

Можно добавить слово с кастомным regex для сложных случаев:

```python
# В БД напрямую или через расширение команды
BlacklistWord(
    word="t.me/",
    normalized_word="tme",  # нормализованная версия
    regex_pattern=r"t\.me/|telegram\.me/|@\w+",  # ловит ссылки и юзернеймы
    action=BlacklistAction.DELETE,
)
```

Тесты проверяют:

- Отсутствие ложных срабатываний на частичных совпадениях

## 📝 Управление через бота

### Список слов
```
/blacklist list
```
Показывает все слова с действиями и статусом.

### Вкл/выкл без удаления
```
/blacklist toggle мат
```

### Принудительное обновление кэша
Черный список кэшируется в Redis на 5 минут. После изменений:
```
/reload_blacklist
```
(только супер-админ)

## ⚠️ Ограничения

1. **Только текстовые сообщения** — фото, стикеры, видео не проверяются
2. **Только группы/супергруппы** — в личке ЧС не работает
3. **Админы игнорируются** — сообщения админов не проверяются
4. **Боты игнорируются** — сообщения других ботов не проверяются
5. **Кэш 5 минут** — изменения вступают в силу через до 5 минут (или `/reload_blacklist`)

## 🛠 Расширение

Для добавления новых типов проверок (например, ссылок, юзернеймов) — расширь `BlacklistAction` и логику в `moderation_service.py`.
